# Workspace 0.3.1

Workspace 0.3.1 is a focused desktop usability update for Files, Chat, and
Space appearance customization. It fixes folder expansion and several visual
regressions without changing Workspace's local-first storage, capability, or
authorization model.

## What is fixed

- **Expandable folders in Files.** Shallow folder-tree responses now preserve
  whether a folder has visible children, so folders such as `manuscript` can be
  expanded and loaded on demand. Hidden Workspace metadata, Office lock files,
  symlinks, and ignored paths continue to follow the existing Files policy.
- **Readable user messages.** Chat chooses dark or light foreground text from
  the actual primary Space color used behind a user message, including custom
  colors, instead of deriving contrast from a blended banner color.
- **Clean dark-mode Markdown.** Fenced code blocks no longer inherit the
  inline-code background in dark mode, removing the appearance that all code
  text is selected.
- **Stop where Send belongs.** While the Assistant is running, the composer
  replaces Send with Stop. The duplicate floating Stop control above the
  transcript is gone.
- **Contained color controls.** Custom color wheels, values, and the paired
  color clear action remain inside their appearance controls without horizontal
  overflow.
- **Working color pairs.** Primary and secondary color pickers update the live
  preview continuously, and the second color is now visibly represented by the
  Classic, Mist, and Pinstripe generated banners as well as the patterns that
  already supported it.

## Security and authorization

This release does not add permissions, connections, credentials, network
destinations, or background authority. A Space remains an ordinary local
folder, and Files continues to exclude executable project configuration and
ignored content according to the existing policy.

## Upgrade notes

No data migration is required. Registered Spaces, Chats, Space appearance
settings, History, Library materials, Skills, Extensions, provider settings,
and ordinary files are preserved by the in-place update.

The secondary color remains a generated-banner treatment. Banner choices whose
design is intentionally single-color, `None`, or a custom uploaded image do not
necessarily display both colors.

## Release verification

The release gates include TypeScript checks, the full test suite, dependency
audit, renderer production build, native Pi preflight, the real-Electron
restricted-app probe, Windows installer construction, packaged-layout and
updater-metadata verification, and tagged cloud publication from the pushed
`main` commit.
