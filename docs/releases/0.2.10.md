# Workspace 0.2.10

Workspace 0.2.10 makes the Apple silicon desktop build feel and behave like a Mac app while preserving the same local-first Spaces, Chats, Capabilities, and restricted-app authority model used on Windows.

## Native macOS experience

- **Mac window chrome.** Workspace uses a hidden-inset native title bar with traffic lights and restrained sidebar vibrancy. Reduce Transparency and an emergency material opt-out receive a theme-matched solid fallback; the work surface stays opaque in every mode.
- **Mac typography and controls.** The renderer uses the macOS system font, system accent color, shortcut glyphs, and native overlay scrollbars. A saved Windows-only Segoe preference migrates to the Mac system default.
- **Native menus.** The Workspace, File, Edit, View, Window, and Help menus now use standard macOS roles for About, Settings, Services, editing, substitutions, speech, emoji, window management, and Quit. Update checks live in the Workspace application menu and use native dialogs.
- **Finder-oriented files.** Registered Space folders participate in Open Recent and represented-filename window behavior. Finder or the Dock can reopen an exactly registered Space, file-tree menus are native on macOS, and file copy uses Finder and Option terminology.
- **Quick Look.** Press Space on a selected file to preview it with Quick Look. Open, reveal, recent-folder, and preview paths remain confined to the registered Space after canonical-path checks.

## Lifecycle and power behavior

Closing the last macOS window no longer tears down the app-owned local API. Accepted Assistant turns and host automations continue, and clicking the Dock icon recreates the window against the same app-lifetime services. Finder and Open Recent also recreate a closed window before routing the Space.

Hidden or occluded Chromium UI may be throttled to save energy; Assistant turns and restricted-app automation work remain owned by the main-process host. Workspace still uses a power-save blocker only while an Assistant turn is active. Explicit **Quit Workspace** performs the normal bounded shutdown.

## Security and authorization

The new native integrations do not broaden Space or restricted-app authority:

- the renderer sends only a Space id for native window identity, and Electron main reloads the canonical name and root from the registered Space catalog;
- Finder/Open Recent accepts only an exact already-registered folder and never registers an unknown path;
- Quick Look and file-menu requests reject absolute, parent-traversal, missing, symlink-escaped, or kind-mismatched targets;
- the native file menu accepts bounded facts, creates all labels in the host, and returns one fixed command id; mutations still use the existing renderer and local-API paths;
- restricted-app notifications remain static, separately granted, enabled-automation-only host notifications. The user-facing wording is now platform-neutral, but rate limits, click revalidation, and teardown rules are unchanged.

## Upgrade notes

The Windows installer and macOS release share version 0.2.10 but remain on separate public GitHub feeds. To update from Mac version 0.2.9, choose **Help → Check for Updates…**. After 0.2.10 is installed, the command follows the native convention at **Workspace → Check for Updates…**. The signed Squirrel.Mac payload replaces the application and relaunches it. Registered Spaces, Chats, History, preferences, Pi state, restricted-app receipts/storage/grants/connections/automations, and ordinary files remain in their existing locations.

The Mac release remains Apple silicon only and requires macOS 12 or newer. The DMG does not edit shell profiles; the bundled `workspace` command is available to Workspace-launched tools, while exposing it to unrelated Terminal sessions remains an explicit user action. System notification and lock-screen settings may suppress or expose reviewed notification text outside the app.
